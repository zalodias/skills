# Explain Code

Explain the selected code in a concise and clear way.

**Goals:**
- Describe intent, design choices, and execution flow
- Treat this as a guided walkthrough or mini-tutorial

## Steps

1. **High-level overview**
   - What problem this code solves
   - What role it plays in the codebase

2. **Important pieces**
   - Point out the most important variables, functions, or components
   - Explain what each one represents in the codebase

3. **Key concepts and patterns**
   - Highlight architectural patterns, idioms, or framework-specific conventions
   - Explain non-obvious language features or APIs

Use simple language. If there are complex parts, break them down further. Reference relevant code samples when explaining features. Include any context about why the code might be written this way.
